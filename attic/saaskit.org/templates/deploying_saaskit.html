Part 5 : Deploying SaaS Kit

To deploy SaaS Kit, navigate to the root dir of SaaS Kit and follow the instructions mentioned below:

  • make -C upstream/ init
  • Edit/Add site-python/local_settings.py according to your enviornment.
  • cd projects/main
  • python manage.py syncdb
  • python manage.py migrate (after syncdb we migrate the necessary tables using South)
  • python manage.py loaddata ../../exampledata.json (This command will overwrite admin account details to admin as login and admin as password, you can change it later to your convenience)
  • python manage.py synccompress (ensure this command works for both the main and user_site)
  • cd media; ln -s /usr/local/lib/python2.6/dist-packages/django/contrib/admin/media admin; the dir could be /var/lib/python-support/python2.6/dist-packages/django/contrib/admin/media
  • python manage.py runserver 8001 (you will have to go up one directory to run this command)
  • OPEN another shell
  • cd projects/user_sites
  • Note: once you add application that are specific to user sites, you need to run 'python manage.py syncdb' for those apps
  • python manage.py synccompress
  • python manage.py runserver
  • Create a new group 'Editors' for templates admin. You can do this by logging in to the admin section(http://yoursite:8001/admin) and create a new Group Editors in Auth>Groups sub-section

When you're comfortable with this section, read part 6 of this tutorial to learn "Getting started with SaaSkit".
